## üìù Submission Process

### 1. Build Your Project
Create something original that showcases Move's capabilities and addresses real needs in the blockchain space. Your project should demonstrate innovation, whether through novel smart contract design, unique educational approaches, or tools that solve genuine developer pain points. Focus on quality over quantity - a well-executed simple project often outperforms complex but incomplete submissions.

### 2. Document Your Work
Comprehensive documentation is crucial for your submission's success. Include a clear project overview explaining what you built and why it matters. Detail all features with descriptions of their functionality and use cases. Provide step-by-step installation instructions that anyone can follow. Add usage examples showing how others can interact with your project. Explain your architecture and technical design decisions. Include testing instructions so reviewers can verify your project works as intended.

### 3. Submit Your Entry

First, fork the Cedra Builders Forge submissions repository and create a new folder for your project. Add all your code, documentation, and supporting materials to this folder. Ensure your README file is comprehensive and follows the provided template. 

Create a pull request to the main repository with a clear title and description. Use the PR template to provide all required information including your project category, brief description, and key features.

After creating your PR, you must notify the team on Telegram - this step is mandatory! Join the Cedra Builders Telegram group and post a message with your project name and PR link. Include your wallet address for prize distribution. This ensures your submission is properly tracked and reviewed.
